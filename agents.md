# Website Authoring Rules

- Follow the local project conventions before introducing new patterns.
- Reuse existing page/component patterns before adding new visual variants.
- Keep edits minimal and targeted to the requested scope.
- Validate rendered output in the real local runtime before finalizing.
- Treat explicit user style/process constraints as mandatory policy for this repo.
- Do not use numbered section headers in `../www` content (for example, avoid `## 1. ...`, `## 2. ...`).
- Do not label sibling sections as `Option A/B/C`; use heading hierarchy with clear titles.
- Do not use list bullets for a single entity. If there is only one item, put it on a new line as plain text/link.
